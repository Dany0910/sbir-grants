# 審查委員觀點

## 概述

本文件整理 SBIR 審查委員的觀點與建議，幫助申請者了解審查的角度。

> 📚 以下內容整理自公開演講與經驗分享，非官方審查標準。

---

## 審查委員在想什麼？

### 審查過程

1. **快速瀏覽**（5 分鐘）
   - 看摘要、看創新點
   - 決定第一印象

2. **詳細閱讀**（20-30 分鐘）
   - 驗證創新性
   - 評估可行性
   - 檢查經費合理性

3. **撰寫評語**
   - 優點與缺點
   - 建議與評分

### 委員的核心問題

審查時，委員心中會問：

1. **這個創新嗎？**
   - 與現有方案有什麼不同？
   - 真的能解決問題嗎？

2. **做得到嗎？**
   - 團隊有能力嗎？
   - 時程合理嗎？

3. **賣得出去嗎？**
   - 有人需要嗎？
   - 商業模式可行嗎？

4. **值得投資嗎？**
   - 效益夠大嗎？
   - 經費花得值嗎？

## 讓委員印象深刻的做法

### ✅ 數據說話

```
好的做法：
「導入本系統後，準確率從 70% 提升至 92%，
  處理時間從 30 分鐘縮短至 3 分鐘。」

不好的做法：
「我們的系統更快更準確。」
```

### ✅ 圖勝於文

- 用架構圖說明技術
- 用流程圖說明方法
- 用表格比較競品
- 用甘特圖呈現時程

### ✅ 客戶驗證

```
好的做法：
「訪談 15 家潛在客戶，12 家確認有此痛點，
  3 家已提供合作意向書（附件）。」

不好的做法：
「我們認為市場需求很大。」
```

### ✅ 團隊實績

```
好的做法：
「主持人曾主導 3 件政府計畫，皆順利結案，
  累計取得 5 件相關專利。」

不好的做法：
「團隊經驗豐富。」
```

## 讓委員扣分的做法

### ❌ 空泛的描述

```
扣分：「本計畫非常創新，採用最先進的技術。」
原因：沒有具體說明，無法評估。
```

### ❌ 誇大不實

```
扣分：「我們是全球第一，沒有競爭者。」
原因：不可信，顯示對市場不了解。
```

### ❌ 資料來源不明

```
扣分：「市場規模達 500 億。」（沒有來源）
原因：無法驗證，降低可信度。
```

### ❌ 格式問題

```
扣分：字體太小、圖表模糊、錯字多
原因：顯示不夠用心，影響專業度。
```

### ❌ 經費不合理

```
扣分：人事費佔 75%，大額設備無說明
原因：不符規範，顯示對規則不了解。
```

## 常見審查評語

### 正向評語

- 「創新點明確，與競品比較清楚」
- 「市場分析紮實，有客戶驗證」
- 「執行計畫具體，查核點明確」
- 「團隊經驗相關，有成功案例」
- 「經費編列合理，說明清楚」

### 負向評語

- 「創新性論述不足，與現有方案差異不明」
- 「市場分析薄弱，缺乏客戶驗證」
- 「時程過於樂觀，可行性存疑」
- 「團隊經歷與計畫關聯性不高」
- 「經費編列比例不符規定」

## 簡報審查技巧

### Phase 1 簡報建議

**時間控制**：
- 簡報：15-20 分鐘
- 問答：10-15 分鐘

**簡報技巧**：
- 每頁一個重點
- 大字大圖，減少文字
- 關鍵數據要突出
- 講重點，不要照念

**問答應對**：
- 聽完問題再回答
- 回答要簡潔
- 不確定可以說「這正是我們要驗證的」
- 承認不足，說明如何克服

### 常見提問

1. **技術問題**
   - 你的演算法如何處理 XXX？
   - 這個技術有什麼風險？

2. **市場問題**
   - 客戶為什麼會選擇你？
   - 你的定價依據是什麼？

3. **團隊問題**
   - 你們之前做過類似的嗎？
   - 如果關鍵人離職怎麼辦？

4. **執行問題**
   - 如果進度落後怎麼辦？
   - 這個時程會不會太趕？

## 評分邏輯

### 委員如何打分

```
90+ 分：這個計畫讓我印象深刻，沒有明顯問題
85-89 分：計畫不錯，有些小問題但不影響大局
81-84 分：計畫可以，符合基本要求
75-80 分：計畫有明顯問題，需要改進
75 分以下：計畫問題太多，不建議核定
```

### 影響評分的因素

**加分項**：
- 有 LOI 或付費客戶
- 有 POC 或試用數據
- 團隊有成功執行案例
- 連結重點政策領域

**扣分項**：
- 資料來源不明
- 格式問題
- 明顯的計算錯誤
- 經費不符規定

## 給申請者的建議

### 準備階段

1. **提早開始**：至少提前 2-3 個月
2. **做好功課**：研究競品、訪談客戶
3. **組好團隊**：找到互補的成員
4. **多次修改**：計畫書至少改 3 遍

### 撰寫階段

1. **站在委員角度**：他們不認識你
2. **數據說話**：每個論述都要有據
3. **圖表優先**：減輕閱讀負擔
4. **請人審閱**：找有經驗的人看

### 審查階段

1. **練習簡報**：至少練 3 遍
2. **準備 Q&A**：預想可能問題
3. **誠實回答**：不要編造
4. **保持冷靜**：被質疑是正常的

---

## 結語

> 「好的計畫書不是寫給自己看的，是寫給不認識你的人看的。」

審查委員每天看很多計畫書，你要讓他們：
- 快速理解你在做什麼
- 相信你的創新是真的
- 相信你做得到
- 相信這值得投資
