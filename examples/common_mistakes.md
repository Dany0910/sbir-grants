# SBIR 常見錯誤與改進建議

## 概述

本文件整理 SBIR 計畫書常見的錯誤，以及如何避免和改進。

---

## 創新性相關錯誤

### 錯誤 1：創新點模糊

**問題**：
```
❌ 「本計畫採用 AI 技術，非常創新」
❌ 「我們的系統比競品更好」
```

**改進**：
```
✅ 「本計畫導入 Transformer 架構，相較傳統 LSTM：
    - 準確率：由 75% 提升至 92%
    - 訓練時間：由 8 小時縮短至 2 小時」
✅ 「與競品 A 比較：
    | 項目 | 競品A | 本計畫 |
    | 速度 | 5秒 | 0.5秒 |
    | 準確率 | 80% | 95% |」
```

### 錯誤 2：創新等於使用新技術

**問題**：
```
❌ 「因為使用 AI，所以很創新」
❌ 「導入區塊鏈技術」
```

**改進**：
```
✅ 說明：使用這個技術解決了什麼問題？
✅ 說明：為什麼非用這個技術不可？
✅ 說明：達成了什麼前人達不到的效果？
```

### 錯誤 3：沒有與競品比較

**問題**：
```
❌ 「市場上沒有競爭者」
❌ 沒有提到任何競品
```

**改進**：
```
✅ 總有替代方案，至少是「人工處理」
✅ 列出 2-3 個主要競爭者
✅ 用表格比較優劣勢
✅ 說明為什麼客戶會選擇你
```

## 市場分析錯誤

### 錯誤 4：市場規模誇大

**問題**：
```
❌ 「全球 AI 市場規模 5,000 億美元」
❌ 用最大的數字來唬人
```

**改進**：
```
✅ 使用 TAM/SAM/SOM 框架
✅ 聚焦到你實際能觸及的市場
✅ 說明計算基礎
✅ 用合理的市佔率估算
```

### 錯誤 5：沒有客戶驗證

**問題**：
```
❌ 「根據我們的判斷，市場需求很大」
❌ 沒有任何客戶訪談或 LOI
```

**改進**：
```
✅ 訪談至少 5-10 家潛在客戶
✅ 記錄客戶的具體回饋
✅ 取得 LOI 或合作意向
✅ 若有 POC，呈現試用結果
```

### 錯誤 6：忽略競爭分析

**問題**：
```
❌ 「我們沒有競爭者」
❌ 競爭分析只有一段話
```

**改進**：
```
✅ 列出直接競爭者、間接競爭者
✅ 分析各競爭者的優劣勢
✅ 說明你的差異化定位
✅ 說明競爭優勢的可持續性
```

## 執行計畫錯誤

### 錯誤 7：時程過於樂觀

**問題**：
```
❌ 6 個月完成 Phase 2 所有工作
❌ 沒有預留緩衝時間
❌ 低估測試與修正時間
```

**改進**：
```
✅ 根據實際經驗估算時間
✅ 預留 20-30% 緩衝
✅ 測試階段時間充裕
✅ 考慮外部因素（法規、合作方）
```

### 錯誤 8：工作項目太籠統

**問題**：
```
❌ 「WP1：開發系統」
❌ 「WP2：測試」
❌ 沒有子項目說明
```

**改進**：
```
✅ 分解到可執行的層級
✅ 每個 WP 有具體產出
✅ 指派負責人
✅ 設定可量化的驗收標準
```

### 錯誤 9：查核點不具體

**問題**：
```
❌ 「CP1：完成第一階段」
❌ 沒有具體的驗收標準
```

**改進**：
```
✅ CP1 (M6)：核心演算法開發完成
   - 產出：演算法模組
   - 驗收標準：準確率 ≥ 85%
   - 證明方式：測試報告
```

## 經費編列錯誤

### 錯誤 10：人事費比例過高

**問題**：
```
❌ 人事費佔 70%（超過 60% 上限）
❌ 人力配置與工作量不符
```

**改進**：
```
✅ 控制人事費在 60% 以內
✅ 說明每個人的投入比例
✅ 確保人力與工作量相符
```

### 錯誤 11：大額設備沒有說明

**問題**：
```
❌ 「設備費：200 萬」（沒有任何說明）
❌ 沒有規格、沒有報價
```

**改進**：
```
✅ 說明設備規格
✅ 說明為什麼需要
✅ 說明現有設備為何不足
✅ 200 萬以上需提供比價
```

### 錯誤 12：費用與工作不對應

**問題**：
```
❌ 編列「國外差旅費」但計畫書沒提到出國
❌ 編列「委外」但沒說明委外內容
```

**改進**：
```
✅ 每筆費用都對應到計畫書內容
✅ 計畫書提到的資源都要編列經費
```

## 團隊介紹錯誤

### 錯誤 13：團隊介紹不相關

**問題**：
```
❌ 列了很多學歷但與計畫無關
❌ 沒說明每個人在計畫中的角色
```

**改進**：
```
✅ 聚焦與計畫相關的經歷
✅ 說明在計畫中的具體職責
✅ 呈現團隊的互補性
```

### 錯誤 14：缺少關鍵能力

**問題**：
```
❌ 技術團隊沒有業務能力
❌ 沒有產業經驗
```

**改進**：
```
✅ 加入產業顧問或業務人員
✅ 說明如何補足不足
✅ 說明合作夥伴的支援
```

## 商業化錯誤

### 錯誤 15：商業模式不清

**問題**：
```
❌ 「之後會賣給客戶」
❌ 沒有說明怎麼賣
```

**改進**：
```
✅ 說明營收模式（訂閱/銷售/服務）
✅ 說明定價策略
✅ 說明銷售通路
✅ 說明客戶開發方式
```

### 錯誤 16：營收預估不合理

**問題**：
```
❌ Y1 營收 5 億（太誇張）
❌ 沒有計算基礎
```

**改進**：
```
✅ 保守估算
✅ 說明計算基礎（客戶數 × 客單價）
✅ 分年度逐步成長
```

## 格式與呈現錯誤

### 錯誤 17：超過頁數限制

**問題**：
```
❌ 計畫書超過規定頁數
❌ 字體太小想塞更多內容
```

**改進**：
```
✅ 遵守頁數限制
✅ 使用規定字體大小
✅ 精簡內容而非縮小字體
```

### 錯誤 18：格式不專業

**問題**：
```
❌ 圖表模糊
❌ 排版雜亂
❌ 錯字很多
```

**改進**：
```
✅ 圖表清晰高解析度
✅ 排版整齊一致
✅ 多次校對
✅ 請他人協助檢查
```
