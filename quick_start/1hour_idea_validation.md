# 1 小時構想驗證

> 快速驗證您的 SBIR 構想是否值得投入資源撰寫計畫書

---

## 使用說明

- 總時間：約 60 分鐘
- 需準備：筆記本、計算機
- 完成後可判斷構想成熟度

---

## Part 1: 創新點驗證（15 分鐘）

### 步驟 1：寫下創新點（5 分鐘）

用一句話描述您的創新：

```
我們的創新是：_______________________________
```

**範例**：
- ✅ 好：「使用 AI 預測庫存需求，準確率提升至 90%+」
- ❌ 不好：「一個很創新的系統」

### 步驟 2：與現有方案比較（10 分鐘）

列出 3 個現有方案，並比較：

| 方案 | 優點 | 缺點 | 我們的優勢 |
|------|------|------|-----------|
| 方案 A |  |  |  |
| 方案 B |  |  |  |
| 方案 C |  |  |  |

**檢查點**：
- [ ] 能列出 3 個現有方案
- [ ] 我們的優勢明確且可量化

---

## Part 2: 客戶驗證（20 分鐘）

### 步驟 3：定義目標客戶（5 分鐘）

回答以下問題：

1. **誰會買**：_______________________________
2. **為什麼買**：_______________________________
3. **願付多少**：_______________________________

### 步驟 4：快速訪談（15 分鐘）

打電話或傳訊息給 3-5 位潛在客戶，問：

1. 您目前如何處理 [問題]？
2. 這個問題有多痛？（1-10 分）
3. 如果有解決方案，您願意付多少？

**記錄結果**：

| 客戶 | 痛點分數 | 願付價格 | 備註 |
|------|----------|----------|------|
| 客戶 A |  |  |  |
| 客戶 B |  |  |  |
| 客戶 C |  |  |  |

**檢查點**：
- [ ] 至少 3 位客戶確認有此痛點
- [ ] 平均痛點分數 ≥ 7
- [ ] 有明確的價格接受度

---

## Part 3: 可行性檢查（15 分鐘）

### 步驟 5：技術可行性（10 分鐘）

回答以下問題：

1. **技術路徑清楚嗎**：□ 是 □ 否
2. **團隊有能力做嗎**：□ 是 □ 否
3. **需要多少時間**：_______________________________
4. **主要風險是什麼**：_______________________________
5. **有對策嗎**：□ 是 □ 否

### 步驟 6：資源檢查（5 分鐘）

估算需要的資源：

- **人力**：_____ 人 × _____ 個月
- **經費**：約 _____ 萬
- **時間**：約 _____ 個月

**檢查點**：
- [ ] 技術路徑清楚
- [ ] 團隊有能力
- [ ] 資源需求合理

---

## Part 4: 市場規模估算（10 分鐘）

### 步驟 7：快速估算（10 分鐘）

使用簡化公式：

```
TAM = 目標產業總產值
SAM = TAM × 適用比例（通常 10-30%）
SOM = SAM × 市佔率目標（第 1 年通常 0.5-2%）
```

**範例**：
```
TAM = 500 億（台灣製造業 IT 支出）
SAM = 500 億 × 20% = 100 億
SOM (第 1 年) = 100 億 × 0.5% = 5,000 萬
```

**您的估算**：
```
TAM = _______________________________
SAM = _______________________________
SOM (第 1 年) = _______________________________
```

**檢查點**：
- [ ] 市場規模合理（不過度樂觀）
- [ ] 有數據來源支持

---

## 下一步行動

### 🟢 值得投入（≥ 80 分）

**立即行動**：
1. 使用 `pre_application_checklist.md`（完整檢核）
2. 開始撰寫計畫書
3. 參考 `templates/phase1_proposal.md`

### 🟡 需補強（60-79 分）

**補強建議**：

| 弱項 | 改進方式 | 時間 |
|------|----------|------|
| 創新點不明確 | 重新定義，參考 `methodology_innovation.md` | 1 週 |
| 客戶驗證不足 | 訪談更多客戶，做 PoC | 2-4 週 |
| 技術不可行 | 做技術驗證，找專家顧問 | 1-2 個月 |

### 🔴 不建議（< 60 分）

**建議**：
- 重新思考構想
- 或先做更深入的市場研究
- 或考慮其他補助方案
