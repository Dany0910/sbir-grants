# 互動式計畫書生成器 - 使用指南

## 🎯 兩種使用方式

### 方式 A：Claude Pro 用戶（推薦）

使用 Claude Projects 功能，體驗最佳。

#### 設定步驟

1. **創建新的 Project**
   - 在 Claude Desktop 中點擊「Projects」
   - 點擊「Create Project」
   - 名稱：「我的 SBIR Phase 1 計畫書」

2. **上傳知識文件**
   - 上傳 `proposal_generator/questions.json`
   - 上傳 `proposal_generator/template_phase1.md`
   - （可選）上傳公司簡介、技術文件等

3. **設定 Custom Instructions**

複製以下內容到 Project 的 Custom Instructions：

```
你是 SBIR Phase 1 計畫書撰寫助手。

當用戶說「開始生成計畫書」時：
1. 載入 questions.json 中的 25 個問題
2. 逐題詢問用戶，記錄所有答案
3. 顯示進度（例如：第 5/25 題）
4. 所有問題回答完畢後，使用 template_phase1.md 範本
5. 將用戶的答案填入範本中的 {{變數}}
6. 生成完整的計畫書

規則：
- 一次只問一個問題
- 等用戶回答後再問下一題
- 如果用戶想修改之前的答案，允許修改
- 保持友善、專業的語氣
- 提供範例幫助用戶理解問題
```

4. **開始使用**

在 Project 中輸入：
```
開始生成 Phase 1 計畫書
```

Claude 會開始逐題詢問，完成後自動生成計畫書！

---

### 方式 B：免費用戶

使用 MCP Server 的狀態管理功能。

#### 使用步驟

1. **開始生成**

```
開始生成 SBIR Phase 1 計畫書
```

2. **回答問題**

Claude 會逐題詢問，例如：
```
Claude: 第 1 題（共 25 題）
請問您的公司全名是？

您: 台灣創新科技股份有限公司

Claude: ✅ 已保存

第 2 題（共 25 題）
請問您的公司主要產業是？
A) 機械
B) 化工/材料
...
```

3. **中斷與繼續**

如果需要中斷：
```
暫停，我稍後繼續
```

下次繼續：
```
繼續生成計畫書
```

4. **查看進度**

```
查看計畫書生成進度
```

5. **生成計畫書**

完成所有問題後：
```
生成計畫書
```

## 🎯 使用技巧

### 1. 準備好資料

開始前準備：
- 公司基本資料
- 問題描述
- 解決方案說明
- 團隊資訊
- 預算規劃

### 2. 詳細回答

越詳細的回答，生成的計畫書品質越高。

**不好的回答**：
```
問：請描述您要解決的核心問題
答：工廠效率低
```

**好的回答**：
```
問：請描述您要解決的核心問題
答：傳統製造業缺乏即時生產數據監控，導致良率僅有 85%，
每月因品質問題損失約 50 萬元，且無法及時發現生產異常。
```

### 3. 使用量化數據

盡量提供具體數字：
- 市場規模
- 效益提升百分比
- 成本降低金額
- 時間節省

### 4. 分段完成

不需要一次完成，可以分多次：
- 第一次：回答基本資訊和問題陳述
- 第二次：回答創新構想和市場分析
- 第三次：回答可行性和團隊
- 第四次：回答經費規劃

## 🆘 常見問題

### Q: 我可以修改生成的計畫書嗎？

A: 可以！生成後您可以：
- 要求修改特定章節
- 重新回答某些問題後重新生成
- 複製到 Word 後自行編輯

### Q: 生成的計畫書可以直接送件嗎？

A: 建議作為初稿，還需要：
- 人工審閱和優化
- 補充具體數據
- 調整語氣和用詞
- 使用檢核清單確認完整性

### Q: 如果我不知道某個問題的答案？

A: 可以：
- 跳過非必填問題
- 填寫「待補充」
- 先填概略答案，之後修改

### Q: Pro 版和免費版有什麼差別？

A: 
- **Pro 版（Projects）**：
  - ✅ 自動記憶所有答案
  - ✅ 可以上傳文件輔助
  - ✅ 隨時修改和優化
  - ✅ 體驗最佳

- **免費版（MCP Server）**：
  - ✅ 所有功能都能用
  - ⚠️ 需要手動保存進度
  - ⚠️ 修改較不方便
