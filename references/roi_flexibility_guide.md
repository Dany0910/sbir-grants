# ROI/ROAS 彈性調整指引

## 基本原則

**保守標準**：補助金額 x 3 倍
**理想目標**：補助金額 x 5 倍

**但這不是死規定！** 實際倍數會根據多種因素綜合調整。

---

## 影響 ROAS 倍數的關鍵因素

### 1. 補助金額規模

| 補助金額 | 建議 ROAS | 原因 |
|---------|----------|------|
| 50-100 萬 | 5-8 倍 | 金額小，期待高報酬 |
| 100-300 萬 | 4-6 倍 | 標準範圍 |
| 300-600 萬 | 3-5 倍 | 金額大，可接受較低倍數 |
| 600-1,200 萬 | 2.5-4 倍 | Phase 2 大型計畫 |

**邏輯**：
- 補助 100 萬 → 期待 500 萬產值（5 倍）
- 補助 1,000 萬 → 期待 3,000 萬產值（3 倍）也可接受
- 因為絕對金額已經很大（3,000 萬 vs 500 萬）

### 2. 計畫類型

| 計畫類型 | ROAS 彈性 | 說明 |
|---------|----------|------|
| 技術研發 | 可較低 | 風險高，但技術價值大 |
| 產品開發 | 標準 | 商業化明確 |
| 服務創新 | 可較高 | 擴展快，成本低 |
| 軟體/數位 | 應較高 | 邊際成本低 |

### 3. 公司規模與營收

| 公司年營收 | ROAS 調整 | 原因 |
|-----------|----------|------|
| < 1,000 萬 | 可較低 | 新創，成長潛力大但風險高 |
| 1,000-5,000 萬 | 標準 | 一般中小企業 |
| 5,000 萬 - 2 億 | 標準偏高 | 有基礎，應有較好執行力 |
| > 2 億 | 應較高 | 資源充足，期待更高效益 |

### 4. 計畫主題

| 主題類型 | ROAS 調整 | 範例 |
|---------|----------|------|
| 前瞻技術 | 可較低 | AI、量子、生技 |
| 成熟技術改良 | 應較高 | 製程優化、自動化 |
| 政策鼓勵產業 | 標準 | 淨零碳排、循環經濟 |
| 紅海市場 | 應較高 | 競爭激烈，需證明優勢 |

## 實務判斷範例

### 範例 1：新創軟體公司

**背景**：
- 公司年營收：500 萬
- 計畫：AI SaaS 平台
- 補助金額：100 萬

**判斷**：
```
基本標準：3-5 倍 = 300-500 萬
調整因素：
+ 軟體業 → +2 倍（邊際成本低）
+ 新創 → -1 倍（風險高）
- 小金額 → +1 倍（期待高報酬）

建議 ROAS：5-7 倍 = 500-700 萬 ✅
```

### 範例 2：中型製造業

**背景**：
- 公司年營收：8,000 萬
- 計畫：智慧化生產系統
- 補助金額：150 萬

**判斷**：
```
基本標準：3-5 倍 = 450-750 萬
調整因素：
+ 製造業 → 標準（3-5 倍）
+ 有規模 → +0.5 倍（執行力強）
+ 成熟技術 → +0.5 倍（風險低）

建議 ROAS：4-6 倍 = 600-900 萬 ✅
```

### 範例 3：大型 Phase 2 計畫

**背景**：
- 公司年營收：2 億
- 計畫：新產品線開發
- 補助金額：1,000 萬

**判斷**：
```
基本標準：3-5 倍 = 3,000-5,000 萬
調整因素：
+ 大金額 → -1 倍（可接受較低倍數）
+ Phase 2 → -0.5 倍（長期計畫）
+ 大公司 → +0.5 倍（資源充足）

建議 ROAS：2.5-4 倍 = 2,500-4,000 萬 ✅
```

## 給使用者的建議

### 建議 1：從保守開始

```
步驟 1：計算基本標準
補助 150 萬 x 3 倍 = 450 萬（最低）
補助 150 萬 x 5 倍 = 750 萬（理想）

步驟 2：評估調整因素
我的產業：製造業（標準）
我的規模：年營收 5,000 萬（標準）
我的主題：成熟技術改良（+0.5 倍）

步驟 3：得出目標
450 萬 → 600 萬（保守）
750 萬 → 900 萬（理想）

我的目標：700 萬（ROAS = 4.7 倍）✅
```

### 建議 2：不要過度糾結倍數

**重點不是「幾倍」，而是「有沒有依據」**

```
❌ 錯誤：「我算出 ROAS 只有 2.8 倍，不夠 3 倍，怎麼辦？」
   → 如果有充分依據，2.8 倍也可能通過

✅ 正確：「我的 ROAS 是 4.5 倍，計算依據是...」
   → 有依據的 4.5 倍 > 沒依據的 10 倍
```

### 建議 3：用「情境分析」展現務實

```
保守情境（機率 60%）：
- ROAS = 3 倍
- 說明：即使在最保守情況下，仍達最低標準

基準情境（機率 30%）：
- ROAS = 5 倍
- 說明：這是我們的目標

樂觀情境（機率 10%）：
- ROAS = 8 倍
- 說明：如果市場反應超出預期

「我們以基準情境作為目標，但即使在保守情境下，
投資效益仍達標準，具備安全邊際」
```

---

## 總結

**三個關鍵原則**：

1. **基本標準是 3-5 倍**
   - 但不是死規定
   - 可根據實際情況調整

2. **重點是「有依據」**
   - 市場規模 x 市佔率
   - 客戶驗證
   - 試算邏輯清楚

3. **展現務實思考**
   - 提供多種情境
   - 用保守估計
   - 說明調整理由

**記住**：
> 審查委員不是在算數學，而是在評估你有沒有想清楚商業模式。
> 一個有充分依據的 3.5 倍，勝過一個沒依據的 10 倍。
